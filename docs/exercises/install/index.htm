Assignment: Installing GIT and Tsugi

In this assignment you will install the GIT source code managment software. GIT allows you to carefully track changes to source code. We can use GIT to store our project in anoy of a number of respositories. GitHub is probably the most popular public repositories. Installing GIT

The first step is to install GIT if you do not already have it installed. If you have GIT installed - you can skip this step.

  • Macintosh: If you have 10.9 or later open a command line and install git using: 
    xcode-select --install
    Then try to run git from the command line to see if it was installed. If that does not work, download and install the GIT client from 
    http://git-scm.com/download/mac
  • Windows: Download and install the GIT client from http://msysgit.github.com/. Note that as you install, you should choose Run Git From Windows Command Prompt rather than the default of Use Git Bash only as seen in this image:

You do not have to follow these instructions exactly as long as one way or another you have a working git on your computer so you can take the next steps.

Configuring GIT

Once you have GIT installed and working from the command line, you need to configure your name and email address with the following commands in the command line / terminal: 

git config --global user.name "John Doe"

git config --global user.email johndoe@example.com
This way you will get credit for any changes you make that you submit to a repository as patches.

In you installed git some other way, just make sure to configure your name and email address before starting to use git.

Installation

The primary task is to get the code checked out into your web server's web documents folder (i.e. htdocs): 

cd /Applications/MAMP/htdocs
git clone https://github.com/csev/tsugi.git
And then follow the rest of the steps inthe installation documentation or video.

Once you copy the file config-dist.php to config.php and edit config.php to set up your global variables and database connection, make sure to set developer mode to TRUE and change the admin password so you can create the tables. Also set the name of the service to be your first name: 

$CFG->servicename = 'Charles';

Tsugi tries to give you hints when it is improperly configured that guide you toward the next step in configuration. For example if you have a good database connection but no database tables defines, you see the following screen:

In Tsugi, you do not need to run "CREATE TABLE" statements. Tsugi has an administration mode where it does all the needed "CREATE TABLE" statements.

Once you have run the "CREATE DATABASE" statements and put the database name, prefix (optional), database id and database pw into config.php - you use the Admin feature (with the password from config.php) and it will create your tables. You should verify that the tables were created using PHPMyAdmin and looking in the "tsugi" database.
